TOPIC 01
Python Variables
Containers that store data in memory
📖 Explanation

Variables are like labeled jars where you store different types of items (numbers, text, etc.). You can change what's inside anytime. Python automatically detects the type — no keyword needed, just use = to assign.

💻 Code Examples
PYTHON 
# Creating variables — no keyword needed, just use =
name       = "Amit"        # String variable
age        = 25            # Integer variable
price      = 99.99         # Float variable
is_student = True          # Boolean variable

# Multiple variables in one line
a, b, c = 1, 2, 3

# Same value to multiple variables
x = y = z = 10

# Printing variables
print(name)   # Output: Amit
print(age)    # Output: 25

# Reassigning
age = 26
print(age)    # Output: 26

# Check variable type
print(type(name))   # <class 'str'>
print(id(age))     # Memory address
🔑 Key Concepts
Concept Explanation Example
Dynamic Typing Python auto-detects type x = 5 → int, x = "hi" → str
Case Sensitivity Upper/lower are different Name ≠ name
Naming Rules Start with letter/_, no spaces ✅ my_var ❌ 2var
Memory Reference Variables point to memory id(var) shows address

TOPIC 02
Data Types & Type Casting
Converting data from one type to another
📖 Explanation

Data Types tell Python what kind of data a variable holds. Type Casting is like converting one currency to another — you change data from one type to another when needed.

💻 Code Examples
PYTHON 
# Core Data Types
age       = 25         # int
height    = 5.9        # float
name      = "Python"   # str
is_active = True       # bool

# TYPE CASTING (Converting)
num_str   = "100"
num_int   = int(num_str)   # 100
num       = 50
num_str   = str(num)       # "50"

pi        = 3.99
pi_int    = int(pi)        # 3  (truncates, not rounds)

x         = 10
x_float   = float(x)      # 10.0

# Boolean to Integer
print(int(True))   # 1
print(int(False))  # 0

# Type Checking
print(type(age))       # <class 'int'>
print(isinstance(age, int))  # True
🔑 Key Concepts
Concept Explanation Example
Implicit Casting Python auto-converts 10 + 5.5 = 15.5
Explicit Casting You manually convert int("100")
Data Loss float→int loses decimal int(3.99) = 3
Invalid Casting Can't convert non-numeric string int("hello") → Error

TOPIC 03
Strings
Text manipulation — immutable sequences of characters
📖 Explanation

Strings are sequences of characters enclosed in quotes. They are immutable — once created they cannot be changed. Python provides rich methods for slicing, joining, searching, and transforming strings.

💻 Code Examples
PYTHON 
# Creating Strings
single     = 'Hello'
double     = "World"
multi_line = """This is
a multi-line string"""

# Concatenation & Repetition
full_name = "Amit" + " " + "Kumar"
print("Ha" * 3)            # "HaHaHa"

# Indexing & Slicing
text = "Python Programming"
print(text[0])              # 'P'
print(text[-1])             # 'g'
print(text[0:6])            # "Python"
print(text[::2])            # every 2nd char
print(text[::-1])           # reversed

# Essential String Methods
s = "  Hello Python World  "
print(s.upper())            # "  HELLO PYTHON WORLD  "
print(s.lower())            # "  hello python world  "
print(s.strip())            # "Hello Python World"
print(s.find("Python"))    # 9
print(s.replace("Python", "Java"))
words = s.strip().split()   # ['Hello', 'Python', 'World']

# Joining
result = "-".join(["Hello", "World"])  # "Hello-World"

# f-strings (modern formatting)
name = "Amit"; age = 25
print(f"My name is {name} and I am {age} years old")

# Checking content
print("123".isdigit())    # True
print("abc".isalpha())    # True
print("Python" in text)  # True
🔑 Key Concepts
Concept Explanation Example
Immutable Cannot change individual characters s[0] = 'a' → Error
Escape Chars Special characters with backslash \n newline, \t tab
String Methods Return new strings (original unchanged) upper(), strip()
Slicing Extract portion of string [start:stop:step]

TOPIC 04
Lists
Ordered, mutable collections — your Python shopping cart
📖 Explanation

Lists are ordered, mutable collections that can hold any type of data. Think of them as a shopping list — you can add, remove, or change items anytime. They grow and shrink automatically.

💻 Code Examples
PYTHON 
# Creating Lists
numbers    = [1, 2, 3, 4, 5]
mixed      = [1, "Hello", 3.14, True]
range_list = list(range(5))      # [0, 1, 2, 3, 4]

# Accessing
fruits = ['apple', 'banana', 'cherry', 'date']
print(fruits[0])     # 'apple'
print(fruits[-1])    # 'date'
print(fruits[1:3])   # ['banana', 'cherry']

numbers.insert(1, 10)       # Insert at index
numbers.extend([5, 6])      # Add multiple

# Removing elements
numbers.remove(10)          # Remove first occurrence
popped = numbers.pop()       # Remove & return last
numbers.pop(2)               # Remove at index

# Other Methods
numbers = [3, 1, 4, 1, 5, 9]
numbers.sort()               # [1, 1, 3, 4, 5, 9]
numbers.sort(reverse=True)  # [9, 5, 4, 3, 1, 1]
print(numbers.count(1))     # 2
print(len(numbers))          # 6

# List Comprehension
squares = [x**2 for x in range(1, 6)]         # [1, 4, 9, 16, 25]
evens   = [x for x in range(1, 11) if x%2==0]  # [2, 4, 6, 8, 10]

# Enumerate (pythonic looping with index)
for index, fruit in enumerate(fruits):
    print(f"{index}: {fruit}")

TOPIC 05
Conditional Statements
Decision making — if, elif, else
📖 Explanation

Conditional statements help your program make decisions based on conditions. Like a traffic light — if green go; if red stop. Python uses indentation (4 spaces) to define code blocks.

💻 Code Examples
PYTHON 
# if-elif-else
score = 85
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"
print(f"Grade: {grade}")   # B

# Logical Operators
age, citizen = 25, True
if age >= 18 and citizen:
    print("Eligible to vote")

day = "Sunday"
if day == "Saturday" or day == "Sunday":
    print("Weekend!")

# Ternary (one-line if-else)
status = "Adult" if age >= 18 else "Minor"

# Membership check
fruits = ['apple', 'banana']
if 'apple' in fruits:
    print("Found!")

# Nested if
age, has_license = 25, True
if age >= 18:
    if has_license:
        print("You can drive")
    else:
        print("Get a license first")
else:
    print("Too young to drive")

TOPIC 06
Loops — For & While
Repeat code efficiently — break, continue, else
📖 Explanation

Loops allow you to repeat code multiple times. for loop iterates over sequences (known count). while loop runs until a condition becomes false (unknown count).

💻 Code Examples
PYTHON 
# ── FOR LOOP ──
for i in range(5):
    print(i)           # 0 1 2 3 4

for i in range(1, 11, 2):
    print(i)           # 1 3 5 7 9

# Loop over list
for fruit in ['apple', 'banana']:
    print(fruit)

# Loop over dict
student = {'name': 'Amit', 'age': 25}
for key, value in student.items():
    print(f"{key}: {value}")

# Zip multiple lists
names = ['Amit', 'Raj']; ages = [25, 30]
for name, age in zip(names, ages):
    print(f"{name} is {age}")

# ── WHILE LOOP ──
count = 1
while count <= 5:
    print(count)
    count += 1

# ── LOOP CONTROL ──
for i in range(10):
    if i == 5: break     # Exit loop
    if i % 2 == 0: continue  # Skip iteration
    print(i)

# Nested loops
for i in range(1, 4):
    for j in range(1, 4):
        print(f"{i} x {j} = {i*j}")

# else with loop (runs if no break)
for i in range(5):
    print(i)
else:
    print("Loop completed")

TOPIC 07
Functions
Reusable blocks of code — def, return, lambda, recursion
📖 Explanation

Functions are reusable blocks of code that perform specific tasks. Like a recipe — once written, you can use it many times with different inputs. They help keep code DRY (Don't Repeat Yourself).

💻 Code Examples
PYTHON 
# Basic function
def greet(name="Guest"):
    print(f"Hello {name}!")

greet()            # Hello Guest!
greet("Amit")      # Hello Amit!

# With return
def add(a, b):
    return a + b

result = add(5, 3)  # 8

# *args — variable positional arguments
def sum_all(*numbers):
    return sum(numbers)

sum_all(1, 2, 3)       # 6
sum_all(1, 2, 3, 4, 5) # 15

# **kwargs — variable keyword arguments
def print_info(**kwargs):
    for k, v in kwargs.items():
        print(f"{k}: {v}")

print_info(name="Amit", age=25)

# Lambda (anonymous one-line function)
square = lambda x: x ** 2
print(square(5))           # 25

nums = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x**2, nums))
evens   = list(filter(lambda x: x%2==0, nums))

# Recursion (function calling itself)
def factorial(n):
    if n == 0 or n == 1: return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120

# Global vs Local scope
counter = 0
def increment():
    global counter
    counter += 1
increment(); print(counter)  # 1

def calculator(a, b, op):
    ops = {
        'add':      lambda: a + b,
        'subtract': lambda: a - b,
        'multiply': lambda: a * b,
        'divide':   lambda: a/b if b != 0 else "Cannot divide by zero"

    fib = [0, 1]
    for _ in range(2, n):
        fib.append(fib[-1] + fib[-2])

TOPIC 09
Encapsulation
Data hiding — public, protected, private
📖 Explanation

Encapsulation means wrapping data and methods together and restricting direct access. Like a capsule — you only see what's necessary. Python uses naming conventions: self._var (protected), self.__var (private).

💻 Code Examples
PYTHON 
class BankAccount:
    def __init__(self, acc, name, bal=0):
        self.account_number  = acc         # Public
        self.holder_name     = name        # Public
        self.__balance       = bal         # Private (double underscore)
        self.__history       = []          # Private

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            self.__history.append(f"+ ₹{amount}")
            print(f"Deposited. Balance: ₹{self.__balance}")

    def get_balance(self):       # Getter
        return self.__balance

    def show_history(self):
        for tx in self.__history:
            print(f"  {tx}")

# account.__balance  ← AttributeError!
# account.get_balance()  ← Correct way

# Property decorator (Pythonic getter/setter)
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def celsius(self):              # Getter
        return self._celsius

    @celsius.setter
    def celsius(self, value):        # Setter with validation
        if value < -273.15:
            print("Below absolute zero!")
        else:
            self._celsius = value

    @property
    def fahrenheit(self):           # Computed property
        return (self._celsius * 9/5) + 32

temp = Temperature(25)
print(temp.celsius)     # 25
temp.celsius = 30        # Uses setter
print(temp.fahrenheit)  # 86.0
🔑 Access Modifiers
Modifier Syntax Access Level Use Case
Public self.name Anywhere General data
Protected self._name Class + Subclasses Internal (convention)
Private self.__name Only within class Sensitive data
TOPIC 10
Inheritance
Child class acquires parent's properties — super(), MRO
📖 Explanation

Inheritance allows a child class to inherit properties and methods from a parent class. Like a child inheriting traits from parents — you get existing features and can add or override them. super() calls the parent class.

💻 Code Examples
PYTHON 
# Single Inheritance
class Animal:
    def __init__(self, name):
        self.name = name
    def eat(self): print(f"{self.name} is eating")
    def sleep(self): print(f"{self.name} is sleeping")

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # Call parent constructor
        self.breed = breed

    def eat(self):            # Override parent method
        print(f"{self.name} eats dog food")

    def bark(self):           # New method
        print(f"{self.name} says Woof!")

dog = Dog("Bruno", "Lab")
dog.eat()    # Bruno eats dog food (overridden)
dog.sleep()  # Bruno is sleeping (inherited)
dog.bark()   # Bruno says Woof!

# Multilevel Inheritance
class Vehicle:
    def __init__(self, brand): self.brand = brand
    def start(self): print(f"{self.brand} starting")

class Car(Vehicle):
    def __init__(self, brand, model):
        super().__init__(brand); self.model = model

class ElectricCar(Car):
    def __init__(self, brand, model, battery):
        super().__init__(brand, model); self.battery = battery
    def start(self): print(f"{self.model} starts silently ⚡")

tesla = ElectricCar("Tesla", "Model 3", 75)
tesla.start()

# Check types
print(isinstance(tesla, ElectricCar))  # True
print(isinstance(tesla, Vehicle))      # True (grandparent)
print(issubclass(ElectricCar, Car))    # True
🔑 Inheritance Types
Type Description Example
Single One parent → one child class Dog(Animal)
Multiple Multiple parents class Child(Father, Mother)
Multilevel Chain of inheritance GrandParent→Parent→Child
Hierarchical One parent, multiple children Animal→Dog, Cat, Bird
TOPIC 11
Polymorphism
Same name, different behavior — duck typing, operator overloading
📖 Explanation

Polymorphism means same method name, different behavior. Like a sound() method — a dog barks, a cat meows, a cow moos. Python also supports operator overloading via magic methods like __add__, __str__.

💻 Code Examples
PYTHON 
# Method Overriding
class Animal:
    def make_sound(self): print("Generic sound")

class Dog(Animal):
    def make_sound(self): print("Woof!")

class Cat(Animal):
    def make_sound(self): print("Meow!")

for animal in [Dog(), Cat(), Animal()]:
    animal.make_sound()   # Woof! / Meow! / Generic sound

# Operator Overloading
class Vector:
    def __init__(self, x, y):
        self.x = x; self.y = y

    def __add__(self, other):   # + operator
        return Vector(self.x + other.x, self.y + other.y)

    def __eq__(self, other):    # == operator
        return self.x == other.x and self.y == other.y

    def __str__(self):          # print()
        return f"Vector({self.x}, {self.y})"

v1 = Vector(2, 3); v2 = Vector(1, 4)
print(v1 + v2)        # Vector(3, 7)
print(v1 == v2)       # False

# Abstract Classes (force implementation)
from abc import ABC, abstractmethod

class Payment(ABC):
    @abstractmethod
    def pay(self, amount): pass

class CreditCard(Payment):
    def pay(self, amount):
        print(f"💳 Paying ₹{amount} via Credit Card")

class UPI(Payment):
    def pay(self, amount):
        print(f"📱 Paying ₹{amount} via UPI")

for method in [CreditCard(), UPI()]:
    method.pay(1000)
TOPIC 12
Abstraction
Hide complexity, show only essentials — ABC, @abstractmethod
📖 Explanation

Abstraction means hiding complex implementation and showing only essential features. Like a car's steering wheel — you don't need to know how the engine works to drive. Use ABC and @abstractmethod to enforce this contract.

💻 Code Examples
PYTHON 
from abc import ABC, abstractmethod

class Vehicle(ABC):
    def __init__(self, brand): self.brand = brand

    @abstractmethod
    def start(self): pass       # Must implement in child

    @abstractmethod
    def stop(self): pass

    def get_brand(self):         # Concrete method
        return self.brand

# v = Vehicle("Toyota")  ← TypeError! Can't instantiate abstract

class Car(Vehicle):
    def start(self): print(f"{self.brand} car engine starts 🔑")
    def stop(self):  print(f"{self.brand} car stops ⛔")

class ElectricCar(Vehicle):
    def start(self): print(f"{self.brand} starts silently ⚡")
    def stop(self):  print(f"{self.brand} regen braking 🔋")

for v in [Car("Toyota"), ElectricCar("Tesla")]:
    v.start(); v.stop()

# Real world: Database abstraction
class Database(ABC):
    @abstractmethod
    def connect(self): pass
    @abstractmethod
    def execute(self, query): pass
    def run(self, query):   # Template method
        self.connect(); return self.execute(query)

class MySQL(Database):
    def connect(self): print("Connecting MySQL...")
    def execute(self, q): print(f"MySQL: {q}")

class PostgreSQL(Database):
    def connect(self): print("Connecting Postgres...")
    def execute(self, q): print(f"Postgres: {q}")
